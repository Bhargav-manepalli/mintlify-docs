# Quickstart

## 1) Sync the Qualcomm Linux Yocto BSP

Create a workspace directory and initialize the manifest (replace placeholders with your release values):

```bash
mkdir -p <release> && cd <release>
repo init -u https://github.com/qualcomm-linux/qcom-manifest -b <branch> -m <manifest>.xml
repo sync
```

## 2) Add the QIM Product SDK layer

Clone the layer into `layers/` (use a release tag that matches your manifest):

```bash
cd <release>
git clone https://github.com/qualcomm-linux/meta-qcom-qim-product-sdk -b <release-tag> layers/meta-qcom-qim-product-sdk
```

## 3) Configure the build environment

Typical environment variables:

```bash
export SHELL=/bin/bash
export EXTRALAYERS="meta-qcom-qim-product-sdk"
export MACHINE=<your-machine>
export DISTRO=qcom-wayland
source setup-environment
```

## 4) Build the multimedia image

```bash
bitbake qcom-multimedia-image
```

## 5) Build the SDK artifacts

```bash
bitbake qcom-qim-product-sdk
```

Next: see **Build Image** and **Artifacts** for paths and troubleshooting.
