# Overview

This site documents how to **sync, build, and use** the Yocto layer `meta-qcom-qim-product-sdk`.

## What you get

- A Yocto layer that integrates Qualcomm multimedia and AI components into a Qualcomm Linux BSP build.
- A repeatable build flow to produce:
  - a flashable image (commonly `qcom-multimedia-image`)
  - a set of SDK artifacts (`qcom-qim-product-sdk`)
  - optionally, a standard SDK and an extensible SDK (eSDK)

## Who this is for

- Platform engineers building BSP images
- App developers consuming the SDK/toolchains
- CI owners who need reproducible builds and artifact publishing

- **Layer source**: `meta-qcom-qim-product-sdk` (Yocto metadata)
