# Generate Yocto SDKs

Yocto provides two SDK styles:

- **Standard SDK**: straightforward cross-compilation toolchain
- **Extensible SDK (eSDK)**: adds workflows for more complex development

## Standard SDK

```bash
bitbake -c do_populate_sdk qcom-multimedia-image
```

## eSDK

```bash
bitbake -c do_populate_sdk_ext qcom-multimedia-image
```

## Output location

SDK installers are usually written under:

```text
<release>/build-<distro>/tmp-*/deploy/sdk/
```

## Using the SDK

- Copy the `.sh` installer to your host.
- Run it and select an install prefix.
- Source the environment setup script it generates (typically `environment-setup-*`).
